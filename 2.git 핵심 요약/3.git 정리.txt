*Git
1.효율적으로 백업하기(코드 저장)

3.협업하기

---

*Branch로 알아보는 git이 협업 원리
-작업 단위로 나눈다. -> 각자 작업한다. -> 합친다. 

---

*git init : main(master) branch에 속한다. 

---

*원격 저장소의 협업 : repository 끼리의 상호 작용

*상호 작용의 종류
-원격저장소에 '조회(추가)하기' : git remote(상호작용이라고 생각)
-원격저장소에 '밀어넣기' : git push
-원격저장소에 '갖고 와서 합치기' : git pull
-원격저장소에 '일단 갖고만 오기' : git fetch
-원격저장소에 '복사하기' : git clone

*참고 : git remote (-v)
-v 옵션 : 단축이름과 URL 같이 보기 가능

---

*원격 저장소 : 여럿이 같이 쓴다. 

*협업의 세 가지 시나리오
-내 로컬 저장소는 변했는데 원격 저장소는 변함 없는 경우 : 코드 작성 후 push하면 된다.
-내 로컬 저장소는 변함 없는데 원격 저장소는 변한 경우 : git pull로 동기화 후 push 한다. 
-내 로컬 저장소도 변했는데 원격 저장소도 변한 경우

---

**내 로컬 저장소도 변했는데 원격 저장소도 변한 경우
*방법 1: pull reqeust(-> merge)
-로컬 : 새 브랜치를 만든다. -> 마음껏 코드 -> 여기서 commit -> 내 코드를 원격저장소에 pull 해주기를 요청 : pull request
*방법 2: rebase
